Lecture 02 – Data and Uncertainty

Motivation: Why Study Data and Uncertainty?

“The goal is not to eliminate uncertainty, but to understand it.”

Every time we flip a coin, roll a die, or run an algorithm, we’re dealing with uncertainty. But where does that uncertainty come from?
Is it a property of the world, our lack of knowledge, or both?

Why This Matters:

In this lecture, we’ll explore how data can help us understand uncertain processes.

We’ll start with simple questions:

  • Is a coin flip really random?
  • How do we model the outcome of a dice roll?
  • Can we trust models built on just three observations?

Growing Complexity

From there, we’ll zoom out:

  • What happens when we simulate many outcomes?
  • How does a histogram relate to the true underlying distribution?
  • What if we want to understand the variability of our estimate?

Models and Data

  • In probability, we represent underlying processes with models — often probability distributions
    (Binomial for coin flips, Multinomial for rolling a dice).

  • These models always rely on assumptions of varying complexity.

  • A key modeling question: How much data is needed? (To model a binomial vs a 3-state multinomial r.v).

  • When model complexity goes up, required amount of data increases : two-sided coin vs. 30-sided dice.

  • Deep learning model is highly flexible and massive amounts of data is needed to train it.

Model Fit to Data

  • Observed data: {Heads, Heads, Tails}

  • Proposed model: Bernoulli random variable
    \[ p(x) = \theta^{x}(1 - \theta)^{1 - x} \]

  • Using maximum likelihood, we estimate: \(\hat{\theta} = 2/3\)

  • The model fits this small dataset perfectly

  • But will it generalize to future coin flips? → Probably not — the sample is too small to reliably estimate \(\theta\)

Model Complexity and Data

  • How do we choose the complexity of a model?
  • It depends on the complexity of the real-world process and the amount of data available.
  • More complex models need more data .
  • Fair model comparisons require out-of-sample evaluation

- Goal: A model that fits current data and generalizes well

Models in the Real World

  • Building models from scratch is often infeasible in complex systems.

  • Example: How would you even begin to model the NCAA tournament? (Human behavior, Team dynamics, Injuries, momentum, randomness…)

  • In such cases, we rely on useful approximations: we can generate new data for comparison.

  • Monte Carlo simulation:Generate synthetic data to explore variability and compare models in complex or uncertain environments

Monte Carlo simulation of complex processes

  • Complex process \(f\) with random input \(X\): i.e. process of interest is \(f(x)\).

  • Simulate data \(X\), apply process \(f\).

  • Calculate some probability or expected value (average).

Monte Carlo simulation for Data Science?

  • Monte Carlo simulation generates fake data. How is this useful?

  • If \(f\) is fairly accurate, we have ground truth.

  • We learn \(\hat{f}\) from (fake) data.

  • We can compare \(\hat{f}\) to \(f\) without worrying about data availability.

Consistency of the sample mean: dice example

  • Suppose average of \(n\)-throws is a complex phenomenon we want to study

  • What is expected value of sample average of \(n\)-observations? Variance?

  • How can we approximate distribution of sample average of \(n\)-observations?

  • Can be calculated theoretically, but we will use simulation to calculate the properties.

  • Suppose one dice throw can be simulated reasonably well using 100 observations. How many observations do we need in order to simulate 5 independent throws of the same dice (iid)?

Simulating dice throws

  • Each fair dice is a random variable \(p_X(x)=1/6\) for all \(x\in\{1,2,3,4,5,6\}\)

  • True value of \(E(X)=\mu=3.5\) and \(\text{Var}(X) = \sigma^2 = 2.917\)

Simulating multiple dice throws (random sample)

  • \(X_1,X_2,\dots,X_n\) is a random sample of size \(n\) from a discrete pdf \(p_X(x)\)

  • Sampling distribution gives \(E(\hat\mu_n)=\mu\) and \[\text{Var}(\hat\mu_n) = \text{Var}\left(\frac{1}{n}\sum_{i=1}^n X_i \right) = \frac{1}{n^2} \sum_{i=1}^n \text{Var}(X_i)= \frac{\sigma^2}{n}\]

  • Can we verify \(E(\hat\mu_n)\) and \(\text{Var}(\hat\mu_n)\) from simulation?

  • Can \(\hat\mu_n\) we verify more complex phenomenon from simulation?
    e.g. does \(\hat\mu\) look like a consistent estimator for \(\mu\)?

More complex phenomenon: Effect of Sample Size \(n\)

Let’s veryfiy Chebyshev’s inequality: \[ \Pr(|\hat\mu_n - \mu| < \epsilon) > 1 - \frac{\text{Var}(\hat\mu_n)}{\epsilon^2} = 1- \frac{\sigma^2}{n\epsilon^2} \]

  • Plug-in variance of dice throws: \(\sigma^2=2.917\)

  • Choose \(\epsilon = 1\)

  • Create dice-throwing function

95% bound according to Chebyshev’s inequality

We can build on the result by showing where 95% bound according to Chebyshev’s inequality.

For illustration, choosing \(95\%\) probability and \(n=50\) (also plug in \(\sigma^2\)), we would get as \(\epsilon\): \[ 1- \frac{2.917}{50\cdot \epsilon^2} = 0.95\\ \epsilon = \sqrt{\frac{2.917}{0.05\cdot 50}}\]

Following computes error threshold \(\epsilon\) guaranteed by Chebyshev’s threshold.

Estimating the CDF from Data

  • Suppose \(Y\sim f(y)\), and \(f(y)\) is the probability density function
  • \(f(y)\) analytically defines behavior or \(Y\)

Data of \(Y\): \(y_1, y_2, \dots\)

  • Each \(y_i\) is one (random) observation of \(Y\)
  • In practice, the true distribution, \(f(y)\), is not known
  • However, we can estimate the empirical distribution, \(\hat f(y)\), from data
  • Histogram \(\hat f(y)\) and equivalently its CDF, \(\hat F(y)\), is the empirical distribution of \(Y\)
  • Recall from 120B definition of CDF: \(F(y) = P(Y\leq y) = \int_{-\infty}^y f(z)\, dz\)
Code 
def ecdf(data):
    y_ord = np.sort(data)
    n = y_ord.size
    Fhat = np.arange(1, n+1) / n
    
    return y_ord, Fhat

Empirical CDF

  • Sort data \(D\) and let order statistic of \(y_i\) be \(y_{(i)}\)

  • We can approximate \(F(y)\) with \(\hat F(y)\) since \[\begin{aligned} &\hat P(Y< y_{(1)}) = 0\\ \hat F(y_{(1)}) = &\hat P(Y\leq y_{(1)}) = 1/n\\ \hat F(y_{(2)}) = &\hat P(Y\leq y_{(2)}) = 2/n\\ &\vdots \\ \hat F(y_{(n)}) = &\hat P(Y\leq y_{(n)}) = n/n = 1\\ \end{aligned}\]

Method of Distributions

  • Simulate “new data” (bootstrap data) with one dataset: \(D = \{y_1, y_2, \dots, y_n\}\)?

  • Can simulate \(U\sim\text{Uniform}(0,1)\) and apply some operation \(g(U) = Y\sim F(y)\)?

  • Assume invertible \(F\) and non-decreasing \(g\), we can show: \(g(u) = F^{-1}(u)\)

  • \(F\) is unknown, but we have \(\hat F\) (revisit ECDF)

Bootstrap Algorithm

Given dataset \(D\) with \(n\) observations:

  1. For \(i=1,2,\dots,n\)
    1. Sample \(u_i\) from Uniform(0,1)
    2. \(y_i^* = \hat F^{-1}(u_i)\),
  2. Return \(D^* = y^*_1, y^*_2, \dots y^*_n\)

Simpler Bootstrap Algorithm

Given dataset \(D\) with \(n\) observations:

  1. For \(i=1,2,\dots,n\)
    1. Sample a random integer \(i\in [1, n]\)
    2. \(y^*_i = y_{(i)}\)
  2. Return \(D^* = y^*_1, y^*_2, \dots y^*_n\)

Bootstrap Theory

Suppose we draw an iid sample \(Y_1, Y_2, \dots Y_B\) from a distribution \(G\). By the law of large numbers

\[ \bar{Y}=\frac{1}{B} \sum_{j=1}^B Y_j \stackrel{\mathrm{P}}{\longrightarrow} \int y d G(y)=\mathbb{E}(Y) \]

as \(B\rightarrow \infty\). More generally, if \(h\) is any function with finite mean then

\[ \frac{1}{B} \sum_{j=1}^B h\left(Y_j\right) \stackrel{\mathrm{P}}{\longrightarrow} \int h(y) d G(y)=\mathbb{E}(h(Y)) \]

Example: Variance

Let \(h\) be sample variance: \[ \begin{aligned} \frac{1}{B} \sum_{j=1}^B\left(Y_j-\bar{Y}\right)^2 & =\frac{1}{B} \sum_{j=1}^B Y_j^2-\left(\frac{1}{B} \sum_{j=1}^B Y_j\right)^2 \\ & \stackrel{\mathrm{P}}{\longrightarrow} \int y^2 d F(y)-\left(\int y d F(y)\right)^2=\mathbb{V}(Y) . \end{aligned} \] We can use sample variance of simulated values to approximate variance of \(Y\).

Other Statistical Estimators

We can approximate \(\mathbb{V}_{\widehat{F}_n}\left(T_n\right)\) by simulation for some estimator \(T_n\).

  • Real world \(\quad F \Longrightarrow X_1, \ldots, X_n \Longrightarrow T_n=g\left(X_1, \ldots, X_n\right)\)
  • Bootstrap world \(\widehat{F}_n \Longrightarrow X_1^*, \ldots, X_n^* \Longrightarrow T_n^*=g\left(X_1^*, \ldots, X_n^*\right)\)

Bootstrap Confidence Intervals

Let \(\theta = T(F)\) and \(\hat\theta = T(\hat F)\):

  • Normal interval: if \(\hat\theta\) is close to normal, \[ \widehat\theta_n \pm z_{\alpha / 2} {\operatorname{se}}_{\text {boot }} \]

  • Pivotal interval \[ C_n=\left(2 \widehat{\theta}_n-\widehat{\theta}_{1-\alpha / 2}^*, 2 \widehat{\theta}_n-\widehat{\theta}_{\alpha / 2}^*\right) \text {. } \]

  • Percentile interval \[ C_n=\left(\widehat{\theta}_{\alpha / 2}^*, \widehat{\theta}_{1-\alpha / 2}^*\right) \text {. } \]

Why Monte Carlo & Bootstrap Matter in Practice

  • Deep Learning
    ↳ Estimate uncertainty in predictions (e.g., dropout as Bayesian approximation)
    ↳ Simulate data to augment small datasets

  • Image Classification
    ↳ Bootstrap to measure variability in accuracy
    ↳ Monte Carlo to simulate label noise or adversarial examples

  • Recommender Systems
    ↳ Simulate user behavior to stress test models
    ↳ Bootstrap to estimate confidence intervals on predictions

  • APIs & Web Data
    ↳ Resampling helps validate insights from partial/incomplete data

Uncertainty is not just a statistical concept — it’s at the heart of robust, real-world data science.
