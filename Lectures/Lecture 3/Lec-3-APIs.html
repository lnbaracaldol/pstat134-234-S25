Lecture 03 ‚Äì Application Programming Interfaces (APIs)

What is an API?

  • API stands for Application Programming Interface
  • It‚Äôs a way for different software systems to communicate with each other
  • Think of it as a waiter in a restaurant:
    • You (the client) ask the waiter (API) for food (data)
    • The waiter gives the request to the kitchen (server)
    • The kitchen prepares the food and the waiter brings it back to you

Real-Life Examples:

  • Your weather app gets updates from a weather data API
  • Logging in with Google or Facebook? That uses an authentication API
  • Buying online? Payment gateways use APIs to securely process transactions

Why APIs Matter in Data Science?

  • Access real-time data (weather, finance, traffic, etc.)
  • Automate data collection at scale
  • Integrate machine learning models into apps
  • Power tools like chatbots, recommendation engines, and more!

Types of APIs

  • Public APIs
    External developers. Often require API keys. (Twitter API, OpenWeatherMap)

  • Private APIs
    Used within an organization to connect internal services. (Internal microservices)

  • Partner APIs
    Shared with specific partners under agreements. (Payment processing between e-commerce and banks)

  • Composite APIs
    Combine multiple API calls into one.

Understanding API Architectures

APIs come in different architectures, which define how they are structured and how they communicate with other systems.

  • REST (Representational State Transfer) The most widely used architecture today.
    • Examples: GitHub REST API, Twitter API, Spotify API
  • SOAP (Simple Object Access Protocol) A strict, standardized protocol.
    • Examples: Salesforce SOAP API, Banking APIs
  • GraphQL (Graph Query Language) A query language for APIs developed by Facebook.
    • Examples: GitHub GraphQL API, Shopify GraphQL
  • gRPC (Google Remote Procedure Call) A high-performance framework created by Google.
    • Examples: Google Cloud, Netflix microservices

API Protocols: The Language of Software Communication

  • An API protocol is a set of rules that govern how different applications exchange data over a network.

  • These protocols define:

    • üì§ How requests are sent
      (e.g., via HTTP, TCP, or MQTT)
    • üì© How responses are structured
      (e.g., using JSON, XML, or Protocol Buffers)
    • ‚ùó How errors are handled
      (e.g., HTTP status codes, custom error messages)

Comparing API Architectures and their protocols

Architecture Request Message Format Key Advantage Common Use Cases
REST HTTP JSON / XML Simple & flexible Web & mobile APIs
GraphQL HTTP JSON Custom data queries Dynamic UIs, client-specific data
gRPC HTTP/2 Protobuf Fast & efficient Microservices
SOAP HTTP/SMTP XML Secure & standardized Enterprise & financial systems
  • Choose the architecture that fits your performance needs, data complexity, and ecosystem support.

HTTP Requests

HTTP stands for HyperText Transfer Protocol ‚Äî it‚Äôs the foundation of data exchange on the web.

An HTTP request is how a client (like your browser or app) asks a server for something (like data or a webpage).

Key Components of an HTTP Request

Element Description
Method Type of action.
URL Location or endpoint being accessed
Headers Extra info like content type, authentication.
Body Optional data sent (POST/PUT requests, JSON format)

Common HTTP Methods

Method Description Use Case Example
GET Request data from the server Retrieve a user profile
POST Send data to create something on the server Submit a form, add a comment
PUT Send data to update or replace something Edit a user‚Äôs profile
PATCH Update part of a resource Change only a user‚Äôs email
DELETE Remove a resource from the server Delete a user account

These methods are the building blocks of interacting with RESTful APIs.

HTTP Responses

When you make an HTTP request, the server replies with a response.

Key parts of an HTTP response:

  • Status Code: Tells if the request was successful

    • 200 OK: Success
    • 404 Not Found: Resource doesn‚Äôt exist
    • 500 Internal Server Error: Something broke on the server

  • Headers: Metadata about the response (e.g. content type, caching)

  • Body: The actual content (e.g. HTML, JSON, text)

APP for random Golden Retrievers

https://dog.ceo/dog-api/

##goldens.py
from fastapi import FastAPI
from fastapi.responses import HTMLResponse
import requests

app = FastAPI()

@app.get("/", response_class=HTMLResponse)
def golden_retriever_image():
    response = requests.get("https://dog.ceo/api/breed/retriever/golden/images/random")
    data = response.json()
    image_url = data["message"]
    
    html_content = f"""
    <html>
        <head>
            <title>Golden Retriever</title>
        </head>
        <body style="text-align: center; margin-top: 50px;">
            <h1>üê∂ Random Golden Retriever</h1>
            <img src="{image_url}" alt="Golden Retriever" width="500"/>
            <p><a href="/">Get another one</a></p>
        </body>
    </html>
    """
    return HTMLResponse(content=html_content)
