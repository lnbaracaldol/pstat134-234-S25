Advanced bash pipelines

# Extract email addresses, sort, unique, and count
grep -E "\w+@\w+\.\w+" emails.txt | \
  sort | uniq > unique_emails.txt
wc -l unique_emails.txt
  • grep -E "\w+@\w+\.\w+" emails.txt
    • grep -E: runs grep with extended regular expressions.
    • Pattern \w+@\w+\.\w+ matches one or more word characters, an @, one or more word characters, a literal ., then one or more word characters (a basic email format).
    • Reads emails.txt and outputs only lines that match this email regex.
  • wc -l unique_emails.txt
    • wc -l counts the number of lines in unique_emails.txt, effectively giving the count of unique email addresses found.

Advanced bash pipelines

# Replace commas with tabs, then extract 2nd & 4th fields
sed 's/,/   /g' data.csv | \
  awk -F"   " '{ print $2, $4 }' > subset.txt
  • awk -F" " '{ print $2, $4 }'
    • awk is a pattern-driven text processing tool.
    • -F" " sets the input field separator to tab.
    • { print $2, $4 } instructs awk to print the 2nd and 4th fields of each line.

Advanced bash pipelines

# Find all .log files, extract IP addresses, count top 10
find /var/log -name "*.log" -type f | \
  xargs grep -Eo '([0-9]{1,3}\.){3}[0-9]{1,3}' | \
  sort | uniq -c | sort -nr | head -n 10
  • find /var/log -name "*.log" -type f
    • Searches the /var/log directory tree
    • -name "*.log" limits to files ending in .log
    • -type f ensures only regular files are matched
  • | xargs grep -Eo '([0-9]{1,3}\.){3}[0-9]{1,3}'
    • xargs takes each filename and runs grep on it
    • grep -Eo enables Extended regex (E) and outputs only the matched portion (o)
    • Pattern ([0-9]{1,3}\.){3}[0-9]{1,3} matches IPv4 addresses: four groups of 1–3 digits separated by dots

Advanced bash pipelines

  • | uniq -c
    • Counts occurrences of each unique IP address
  • | sort -nr
    • Re-sorts the lines numerically (-n) in reverse order (-r) so highest counts appear first

Static vs Dynamic Websites

Static Sites

  • All content delivered in the initial HTML response
  • Little to no client-side JavaScript
  • Scraping: simple HTTP GET + parse with rvest/BeautifulSoup

Dynamic Sites

  • Content populated or modified by JavaScript after load (AJAX calls, frameworks)
  • Often built as Single-Page Applications (React, Vue, Angular)
  • Scraping: requires executing JS or driving a browser (Chromote, Selenium)

How to Tell If a Site is Dynamic

  1. View Page Source (Right-click > View Page Source)
    • If you can’t find the data in the raw HTML, it’s likely dynamic.
  2. Disable JavaScript and Reload
    • Chrome DevTools → Cmd + Shift + P → “Disable JavaScript”
    • Reload the page → if the content disappears, it’s dynamically rendered.
  3. Look for JavaScript frameworks
    • <div id="root"> or <div id="app">

️ Challenges of Dynamic Web Scraping

  • JavaScript execution required (not handled by base R/Python)
  • Requires setup of headless browsers or browser drivers
  • Elements may only load after user interaction (scrolling, clicking)
  • Websites may block scraping with bot detection
  • Authentication or tokens may be required for content
  • Content may change frequently (e.g., pagination, infinite scroll)
  • Performance: dynamic scraping is slower and more resource-intensive

Live Web Scraping for Dynamic Websites

Modern web applications often rely on JavaScript to fetch and render data after the initial page load. Simple HTTP requests only retrieve the raw HTML skeleton; not the content populated by client‑side scripts.

  • Technical process:

    1. Fetch the initial HTML.
    2. Execute embedded JavaScript via a browser engine (e.g., V8).
      • Runs event loops, resolves AJAX calls, manipulates the DOM.
    3. Simulate user actions (scroll, click) if needed to trigger lazy‑loaded content.
    4. Extract the fully rendered DOM for parsing.

  • Tools like Chromote and Selenium launch a real or headless browser, drive it programmatically, and then capture the updated HTML just as a human would see it.

Python Demo: Selenium

  • Challenge: Many sites use JS frameworks to render data after initial load.
  • Solution: Use Selenium WebDriver to automate a real browser, wait for rendering, then grab page_source.
## Static
import requests
from bs4 import BeautifulSoup

url = "https://quotes.toscrape.com/js/"
resp = requests.get(url)
soup = BeautifulSoup(resp.text, "html.parser")

### Try to grab the quotes statically
static_quotes = [span.get_text(strip=True)
                 for span in soup.select(".quote span.text")]
print("Static (no JS):", static_quotes)

Python Demo: Selenium

### Dynamic (Selenium)
from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# Start a Chrome session (you need chromedriver installed/in PATH)
service = Service()  # or Service("/path/to/chromedriver")
driver = webdriver.Chrome(service=service)

#  Load the page and wait for quotes to render
driver.get("https://quotes.toscrape.com/js/")

# wait until at least one quote span is visible
WebDriverWait(driver, 10).until(
    EC.presence_of_element_located((By.CSS_SELECTOR, ".quote span.text"))
)
